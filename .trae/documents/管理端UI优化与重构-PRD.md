## 1. Product Overview
扫码点餐系统「管理端」UI 优化与重构：统一视觉与交互、提升效率与可维护性。
通过引入 Tailwind CSS v3.4.17 与 shadcn-ui 组件体系，建立可复用的后台设计系统。

## 2. Core Features

### 2.1 User Roles
| 角色 | 注册/开通方式 | 核心权限 |
|------|----------------|----------|
| 管理员 | 由系统预置或由超管创建账号 | 管理门店配置、账号权限、全部业务数据查看与操作 |
| 门店运营 | 管理员创建账号并分配权限 | 订单处理、菜品维护、桌台/二维码维护（受权限控制） |

### 2.2 Feature Module
管理端需求由以下页面组成：
1. **登录页**：账号登录、错误提示、记住登录态。
2. **控制台总览**：后台框架布局（侧边栏/顶部栏）、关键指标概览、快捷入口。
3. **订单管理页**：订单列表、筛选/搜索、订单详情抽屉/弹窗、状态流转（接单/出餐/完成/取消）。
4. **菜品与分类管理页**：分类列表、菜品列表、上下架与价格库存编辑、图片上传入口。
5. **桌台/二维码管理页**：桌台列表、新增/编辑、二维码生成与下载、状态（启用/停用）。
6. **系统设置页**：门店信息与营业设置、打印/支付等基础配置入口、账号与权限管理入口。

## 3. Core Process
**门店运营/管理员主流程**
1) 进入登录页完成登录，进入控制台总览。
2) 在总览查看待处理订单并进入订单管理：筛选“待接单/制作中”等状态。
3) 打开订单详情并执行状态流转（接单→出餐→完成/取消）。
4) 进入菜品与分类管理：维护分类、编辑菜品信息并上下架。
5) 进入桌台/二维码管理：维护桌台并下载二维码用于张贴。
6) 进入系统设置：维护门店信息与基础配置；管理员维护账号权限。

**迁移策略（UI 优化与重构）**
- 以“页面壳 + 设计 tokens + 基础组件”优先：先落地全局布局与 shadcn 基础组件（Button/Input/Dialog/Table/Toast）。
- 以“高频业务先迁移”：优先迁移订单管理页，再迁移菜品管理、桌台管理、系统设置。
- 严格“替换不增项”：重构期间不新增业务功能；仅做样式一致性、交互修复与组件抽象。
- 双轨交付与回滚：按路由/页面逐步切换；保留旧页面入口以便灰度与快速回退。
