# 小程序多机型适配优化指南（rpx + 媒体查询 / 溢出与弹窗规则 / 真机测试模板）

## 1. 目标与适用范围

* 目标：在不同屏幕尺寸、分辨率、刘海/圆角/底部安全区、系统字号设置、输入法弹起等场景下，保持布局稳定、可读性一致、可触控性达标。

* 适用：微信小程序（WXML/WXSS），桌面优先不适用；本规范采用**手机优先**（小程序常规）并通过媒体查询覆盖大屏。

***

## 2. 布局规范：rpx 基线 + 媒体查询分档

### 2.1 rpx 使用原则

1. **尺寸用 rpx，边框用 px**：

   * 间距/宽高/圆角/字号：优先 rpx（随屏幕宽度缩放，适合等比例布局）。

   * 1px 细线：使用 `1px`（避免在高分屏出现粗线）。
2. **避免“全屏绝对定位”**：尽量用 Flex/Grid（微信小程序建议 Flex 为主）。
3. **用容器约束替代硬编码**：

   * 列表：`flex: 1` + `min-height` 而不是写死高度。

   * 卡片：`max-width` 控制在大屏不被拉伸过度。

### 2.2 基准设计与分档策略

* 设计基准：以 **750rpx** 作为设计稿宽度（行业常规）。

* 分档思路：默认样式覆盖绝大多数手机；对**小屏**与**平板/大屏**做覆盖。

建议断点（可按业务微调）：

* 小屏：≤ 360px

* 标准：361–414px

* 大屏手机：415–767px

* 平板：≥ 768px

### 2.3 WXSS 媒体查询写法（示例）

> 将断点样式集中在 `app.wxss` 或 `styles/responsive.wxss`，避免页面到处散落。

```css
/* 默认（标准手机） */
:root, page {
  --space-1: 16rpx;
  --space-2: 24rpx;
  --radius-md: 16rpx;
  --font-body: 28rpx;
  --font-title: 32rpx;
  --btn-h: 88rpx;
}

/* 小屏：降低横向拥挤 */
@media (max-width: 360px) {
  page {
    --space-1: 12rpx;
    --space-2: 20rpx;
    --font-body: 26rpx;
    --btn-h: 84rpx;
  }
}

/* 平板/大屏：限制内容最大宽度 + 提高信息密度 */
@media (min-width: 768px) {
  page {
    --font-body: 30rpx;
    --font-title: 36rpx;
  }

  .container {
    max-width: 900rpx; /* 防止过宽导致阅读跨度过大 */
    margin: 0 auto;
  }
}
```

### 2.4 安全区（刘海/底部 Home Indicator）

* 底部吸底栏（结算条/提交按钮）必须预留安全区：

  * 样式层：优先 `padding-bottom: env(safe-area-inset-bottom);`

  * 逻辑层：用 `wx.getSystemInfoSync().safeArea` 计算兜底（不同机型兼容更稳）。

推荐规则：

* 吸底区域最小内边距：`24rpx`，并额外叠加安全区 inset。

* 全屏弹窗、底部弹窗的内容区高度计算必须扣除安全区（见第 4 节）。

## 3. 字号与按钮触控规范（可读性 + 可点击性）

### 3.1 字号层级（建议）

| 场景          |     建议字号 | 说明        |
| ----------- | -------: | --------- |
| 页面标题 / 弹窗标题 | 32–36rpx | 重要信息，避免过小 |
| 正文 / 表单文字   | 26–30rpx | 默认阅读区间    |
| 辅助信息 / 说明   | 22–26rpx | 需保证可读     |
| 标签/角标       | 20–24rpx | 不小于 20rpx |

规则：

* **正文不建议低于 26rpx**（小屏可降至 24–26rpx，但需真机确认可读）。

* 同一页面字号层级不超过 4 层，避免视觉噪音。

### 3.2 按钮与可点击区域（触控最小尺寸）

| 元素          | 最小可点击高度 | 最小可点击宽度 | 推荐             |
| ----------- | ------: | ------: | -------------- |
| 主按钮（提交/支付）  |   88rpx |  240rpx | 统一按钮高度（利于肌肉记忆） |
| 列表项点击行      |   96rpx |    100% | 行内元素避免“点不到”    |
| 图标按钮（返回/加减） |   80rpx |   80rpx | 图标可小，但点击热区要大   |

规则：

1. **视觉尺寸可以小，触控热区必须达标**：用 `padding` 扩大可点区域。
2. 禁止把多个操作塞到 1 个很小的区域（如紧挨的“编辑/删除”图标）。
3. `disabled` 状态需明显（降低不透明度 + 改变背景/描边），同时保留可读对比度。

## 4. scroll-view 溢出治理规则（列表/表单/弹窗内滚动）

### 4.1 何时使用 scroll-view

* 使用场景：**长列表、弹窗内容区、需要局部滚动**的区域。

* 避免场景：页面本身已可滚动时，再嵌套 `scroll-view`（容易出现“滚不动/穿透/回弹异常”）。

### 4.2 溢出治理（必须项）

1. **给 scroll-view 明确高度**：

   * 推荐：外层容器 `flex: 1`，scroll-view `height: 100%`。

   * 或：用计算高度（例如扣除头部/底部固定区域）。
2. **滚动方向单一**：

   * 纵向列表：只开 `scroll-y`；横向滑动：只开 `scroll-x`。
3. **内容超出策略**：

   * 文本：`overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`（单行）

   * 多行：用业务约束（最多 2 行 + 省略）或展开收起。

### 4.3 scroll-view 在弹窗中的推荐结构（示例）

```xml
<!-- 弹窗容器固定高度；内容区滚动 -->
<view class="modal">
  <view class="modal__header">...</view>
  <scroll-view class="modal__body" scroll-y>
    ...长内容...
  </scroll-view>
  <view class="modal__footer">...</view>
</view>
```

## 5. 弹窗高度与键盘空间规则（防遮挡 / 可操作）

### 5.1 弹窗高度（通用规则）

* **全屏弹窗**：内容区高度 = `windowHeight - safeTop - safeBottom`。

* **底部弹窗（sheet）**：

  * 最大高度：`windowHeight * 0.85`（避免压迫感、保留关闭手势空间）。

  * 默认高度：按内容自适应，但超过最大高度时，**内部内容区使用 scroll-view**。

### 5.2 键盘弹起（表单/搜索/备注输入）

目标：输入框永远可见、提交按钮可触达。

规则（从强到弱的优先级）：

1. **优先使用系统能力让页面上移**：

   * `input/textarea` 开启 `adjust-position="true"`（默认通常为 true，但建议显式写）。

   * 通过 `cursor-spacing` 预留输入框与键盘的间距（建议 16–24px）。
2. **弹窗内输入时**：

   * 弹窗整体不要跟随键盘“强行拉伸”；改为：

     * 保持弹窗最大高度不变；

     * 将 `modal__body` 变为可滚动；

     * 把当前输入项滚动到可视区域（用 `scroll-into-view` 或测量后滚动）。
3. **底部固定按钮（如提交）**：

   * 键盘弹起时：按钮需“贴键盘上沿”或折叠为悬浮小条（两者选一并保持一致）。

### 5.3 键盘空间计算建议（实现提示）

* 监听键盘高度变化（如平台提供的 keyboardHeight 事件/回调）；

* 可视底部空间 = `keyboardHeight + safeAreaBottomInset`；

* 弹窗 footer 的 `bottom` / `padding-bottom` 以该值作为下边距兜底。
