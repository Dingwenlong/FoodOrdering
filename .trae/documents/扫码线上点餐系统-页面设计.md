# 扫码线上点餐系统-页面设计（桌面优先 + 小程序移动端）

## 全局样式（两端通用建议）
- 颜色：背景 #F7F8FA；主色 #1677FF；成功 #52C41A；警告 #FAAD14；危险 #FF4D4F；文本 #1F2329 / 次级 #646A73
- 字体：管理端 14/16/20/24px 四档；小程序 12/14/16/18px 四档；行高 1.4–1.6
- 按钮：主按钮实心主色；次按钮描边；hover/active 管理端加深 8%；禁用灰度
- 卡片：圆角 8px，阴影轻量；列表分割线 1px #EDEEF0
- 交互：所有“提交/状态变更”按钮需二次确认或可撤销提示（占位）

## 小程序端（移动优先）

### 1) 扫码/桌台确认页（pages/scan/index）
- Layout：纵向堆叠（Flex column）；底部固定主按钮；安全区适配。
- Meta：分享标题“扫码点餐”；描述“快速下单”；OG同小程序分享信息。
- Page Structure：顶部门店信息卡 + 桌号确认区 + 异常提示区 + 底部“开始点餐”。
- Sections & Components：
  - 门店卡：门店名/营业状态/联系电话（可拨打）。
  - 桌台信息：桌号大字显示；“重新扫码”入口。
  - 异常态：无效码/过期/门店未营业（仅文案+返回）。

### 2) 点餐页（菜单）（pages/menu/index）
- Layout：左右分栏（左分类窄栏 + 右菜品列表）；右下角悬浮购物车。
- Page Structure：顶部门店+桌号条；主体分类/菜品；底部购物车摘要条。
- Components：
  - 分类栏：当前高亮；支持滑动。
  - 菜品卡：图/名/价/售罄；“+”加购。
  - 规格弹层：规格单选、加料多选占位、数量步进器、确认加入。
  - 购物车浮层：点击摘要条展开。

### 3) 购物车与下单页（pages/cart/index）
- Layout：列表 + 底部结算栏（fixed）。
- Sections：
  - 购物车列表：菜品行（名称/规格/单价/数量/删除）。
  - 备注输入：多行文本（字数限制）。
  - 金额区：小计/服务费占位/合计。
  - 提交按钮：点击后加载态；失败toast；成功跳转订单详情。

### 4) 订单详情页（pages/orderDetail/index）
- Layout：状态头部卡 + 明细列表 + 操作区（根据状态显示）。
- Sections：
  - 状态卡：状态标签、倒计时占位（待支付）。
  - 明细：商品明细、金额、备注。
  - 操作：待支付“去支付”；其他状态“返回继续点餐”占位。

## 管理端（桌面优先，>=1200px 适配）

### 全局布局
- Layout：经典后台三段式：左侧 Sider 导航 + 顶部 Header + 内容 Content（CSS Grid + Flex 混合）。
- Header：门店选择器（单店可隐藏）+ 账号菜单（退出）。
- Content：面包屑（可选）+ 页面标题 + 工具条 + 表格/表单。

### 5) 登录页（/login）
- Meta：title“管理端登录”；description“订单与菜单管理”。
- Layout：居中卡片（max-width 420px），背景浅灰。
- Components：账号/密码输入、登录按钮、错误提示、加载态。

### 6) 订单管理页（/orders）
- Layout：上工具条 + 下表格（全宽）；右侧抽屉展示详情。
- Components：
  - 筛选条：状态Tabs、时间范围、关键字；查询/重置。
  - 表格：订单号、桌号、金额、状态、下单时间、操作。
  - 详情抽屉：明细、备注、状态流；操作按钮（接单/拒单/出餐/完成）。

### 7) 菜单管理页（/menu）
- Layout：左分类列表 + 右菜品表格/卡片；弹窗做新增/编辑。
- Components：
  - 分类区：新增/排序；选中高亮。
  - 菜品区：搜索、上下架开关、价格、售罄标记；图片上传占位。
  - 编辑弹窗：名称、分类、价格、状态、规格占位。

### 8) 门店与桌台二维码页（/store-tables）
- Layout：上门店信息表单 + 下桌台表格（含二维码操作）。
- Components：
  - 门店表单：名称、营业时间、电话、支付配置占位。
  - 桌台表格：桌号、状态、二维码预览/下载、批量生成。

## 响应式规则（管理端）
- >=1200px：默认三段式；表格完整列。
- 768–1199px：Sider可折叠；表格列压缩并启用横向滚动。
- <768px：仅保证可用性（折叠导航+抽屉筛选），但不作为主体验目标。
